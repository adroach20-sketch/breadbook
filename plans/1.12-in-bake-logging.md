# 1.12 In-Bake Logging Phase 2

## Debate: Head Chef vs UX Designer

### Round 1 - Initial Positions

**Head Chef (Data Integrity):**

1. Dough observations (bulk ferment): Feel and smell are the two signals bakers use to decide when to end bulk. Logging creates a feedback loop for the Troubleshooter (2.5). Single-select per check-in - models temporal changes better than multi-select.
2. Shaping step: Log method used + dough feel. Method changes matter for iteration, feel matters for troubleshooting.
3. Proofing step: Fridge timestamp is critical - cold proof duration drives crumb and flavor. Poke test gives beginners concrete options.
4. Off-plan button: Most important feature. Every bake goes off-plan. Reason (structured) + note (freeform) both matter.

**UX Designer (Mobile-First):**

1. Dough observations: 6 feel + 4 smell = 10 pills. Split into two labeled rows. Single-select per check-in.
2. Shaping: Boule / Batard / Rolls / Other + dough feel pills. Two compact rows.
3. Proofing: Big "Into the fridge" button (cold_proof). Poke test pills below.
4. Off-plan: Small trigger link on every step. Opens bottom sheet with reason pills + optional text input.

### Round 2 - Consensus

- Single-select per observation (better temporal data than multi-select)
- Shaping methods: Boule / Batard / Rolls / Other (four pills)
- Off-plan: reason is required (structured), note is optional (freeform, 200 char limit)
- Observation widgets go in position 8 (alongside FoldTracker/RiseCheckIn)
- Off-plan trigger goes below step counter, always visible

### Round 3 - Final Architecture

#### New Event Types
- dough_feel, dough_smell, shaping_method, shaping_feel, fridge_in, poke_test, off_plan

#### New Components
1. DoughObservation.tsx - Feel + Smell pill rows with mini-log
2. ShapingLog.tsx - Method + Feel pill rows
3. ProofingLog.tsx - Fridge button + poke test pills
4. OffPlanSheet.tsx - Bottom sheet with reason pills + text input

#### Implementation Order
1. Types (extend BakeEvent union)
2. DoughObservation
3. ShapingLog
4. ProofingLog
5. OffPlanSheet
6. Wire into StepView + BakeMode
