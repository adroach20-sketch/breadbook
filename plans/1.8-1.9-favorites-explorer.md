# Features 1.8 and 1.9 - Recipe Favorites + Recipe Explorer and Search

## Multi-Perspective Debate

### Round 1: Initial Perspectives

#### Head Chef (Technical Accuracy and Data Integrity)

On 1.8 (Favorites):
- The recipe_saves table needs a composite unique constraint on (user_id, recipe_id). RLS must enforce authenticated users only.
- Recipes currently have string IDs locally but UUIDs in Supabase. Favorites must save against the Supabase UUID. The Recipes page does a title-based merge, so we need Supabase data loaded before favorites render.
- Optimistic UI needs error revert. Silent failures would confuse users.
- Favorites tab is cleaner than re-sorting the All view.
- Save count: client-side aggregation is fine for MVP. Denormalize later.

On 1.9 (Explorer and Search):
- Search should work against the loaded recipe array (client-side). Sufficient for under 100 recipes.
- Filter fields like Technique are derived from step types in JSONB. Client-side filtering works fine at this scale.
- Trending/Most Saved sections should render conditionally based on data availability.
- Starter-aware suggestions require starter data (feature 2.1). Stub the banner only.

#### UX Designer (Mobile-First Flow and Discoverability)

On 1.8 (Favorites):
- Heart icon: top-right of recipe card with absolute positioning over the Link, using stopPropagation.
- On detail page: heart near title in header, large enough for thumb tap.
- Favorites tab should be the LAST tab, not first.
- No re-sorting of All tab. Favorites tab only.

On 1.9 (Explorer):
- Explore tab should feel like discovery with editorial sections and horizontal carousels.
- Filter panel must be collapsible, default collapsed.
- Search bar sticky at top with inspiring placeholder text.
- Explore replaces Recipes in nav. Route stays /recipes for backwards compat.
- Save count: small heart + number in metadata row, only if count > 0.

### Round 2: Critiques and Consensus

1. Heart icon: top-right of recipe card, absolute positioned, stopPropagation. On detail page: near title.
2. Favorites tab: add as LAST tab. No re-sorting of All tab.
3. Explore replaces Recipes in nav. Route stays /recipes.
4. Search + filters: search bar sticky at top, filter panel collapsible.
5. Editorial sections: horizontal scroll carousels, conditionally rendered.
6. Save count: heart + number on cards, only shown if > 0.
7. Stub starter-aware banner (no real data).

### Round 3: Final Architecture

Implementation Order:
1. Migration: 004_recipe_saves.sql
2. Types: extend types for save data
3. Zustand store: favorites.ts with optimistic toggle
4. FavoriteButton component
5. Modify RecipeCard + RecipeDetail
6. SearchBar component
7. FilterPanel component
8. RecipeSection (editorial carousel)
9. Explore page
10. Update Layout.tsx nav + App.tsx routes
