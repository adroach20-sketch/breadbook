# BreadBook Mobile App — Capacitor Plan

> Decided 2026-03-10. Capacitor wraps the existing React web app as a native iOS + Android app.

---

## Why Capacitor (Not React Native, Not PWA-Only)

| Approach | Verdict | Reason |
|---|---|---|
| **PWA only** | Not enough | iOS throttles background timers, PWA push notifications unreliable on iOS |
| **React Native** | Overkill | Complete UI rewrite, 0% code reuse, massive effort for a solo dev |
| **Capacitor** | Right fit | Wraps existing React app, ~95% code reuse, native APIs where needed |

Capacitor runs our existing Vite + React app inside a native WebView, then gives us access to native APIs (push notifications, local notifications, haptics) via plugins. One codebase → web + iOS + Android.

---

## What Capacitor Solves

### 1. Timers That Survive Background / Lock Screen
**Current state:** `useTimer.ts` uses `setInterval` + `endTime` ref + localStorage. Works on page refresh and tab switch, but iOS kills the interval when the app is backgrounded.

**Capacitor fix:** When a timer starts, schedule a `LocalNotification` at the exact end time. The OS handles the notification natively — fires even if the app is killed. When the app returns to foreground, `useTimer` already recalculates from the stored `endTime`, so the UI is correct.

**Code change:** Small addition to `useTimer.ts`:
```typescript
import { Capacitor } from '@capacitor/core'
import { LocalNotifications } from '@capacitor/local-notifications'

// In the start() function, after setting endTimeRef:
if (Capacitor.isNativePlatform()) {
  await LocalNotifications.schedule({
    notifications: [{
      title: 'BreadBook Timer',
      body: "Time's up! Ready for the next step.",
      id: timerNotificationId,
      schedule: { at: new Date(endTimeRef.current) },
      sound: 'timer_done.wav',
    }]
  })
}

// In pause() and reset(), cancel the scheduled notification:
if (Capacitor.isNativePlatform()) {
  await LocalNotifications.cancel({ notifications: [{ id: timerNotificationId }] })
}
```

The web version continues to use browser `Notification` API as it does today. `Capacitor.isNativePlatform()` gates the native code — no impact on web builds.

### 2. Push Notifications for Starter Feeding Reminders
**Current state:** No push notifications. Feeding reminders are planned but not built yet.

**Capacitor fix:** Use `@capacitor/push-notifications` to register for remote push via APNs (iOS) / FCM (Android). Supabase can trigger pushes via Edge Functions or a cron job.

**Architecture:**
```
User sets feeding schedule → Supabase stores schedule
Supabase cron (pg_cron or Edge Function) → checks due reminders
→ sends push via FCM/APNs
→ notification appears on phone even if app is closed
```

This is a Phase 2 feature (after Capacitor is set up and basic app is working).

### 3. App Store Presence
Being in the App Store and Play Store is a discovery and credibility signal. "Download BreadBook" is a more natural call-to-action than "visit breadbook.app and add to home screen."

## Dependencies to Install

```bash
# Core Capacitor
npm install @capacitor/core @capacitor/cli

# Native project generation
npx cap init BreadBook com.breadbook.app --web-dir dist

# Platform targets
npm install @capacitor/ios @capacitor/android
npx cap add ios
npx cap add android

# Plugins we need
npm install @capacitor/local-notifications   # Timer alerts
npm install @capacitor/push-notifications    # Feeding reminders (Phase 2)

npm install @capacitor/haptics               # Tactile feedback
npm install @capacitor/app                   # App lifecycle events
npm install @capacitor/keyboard              # Keyboard behavior on mobile
```

## Build & Deploy Workflow

### Development (Windows — Android only)
```bash
npm run build                 # Build the Vite app
npx cap sync                  # Copy dist/ to native projects + sync plugins
npx cap open android          # Open in Android Studio
# Run on emulator or connected device from Android Studio
```

**Live reload during dev:**
```bash
npm run dev                   # Start Vite dev server
# In capacitor.config.ts, uncomment server.url = 'http://YOUR_LOCAL_IP:5173'
npx cap run android --livereload --external
```

### Development (Mac — iOS)
```bash
npm run build
npx cap sync
npx cap open ios              # Open in Xcode
# Run on simulator or connected device from Xcode
```

### Production Build
```bash
npm run build                 # Production Vite build
npx cap sync                  # Sync to native projects

# Android: Build signed APK/AAB in Android Studio
# iOS: Archive and upload to App Store Connect via Xcode
```

### CI/CD (future)
- GitHub Actions can build Android (no Mac needed)
- iOS builds require macOS runner (GitHub Actions has them, or use a Mac)

## Code Changes Required

### Minimal changes — most of the app works as-is.

#### 1. `useTimer.ts` — Add native local notification scheduling
- On `start()`: schedule a `LocalNotification` at `endTimeRef.current`
- On `pause()` / `reset()`: cancel the scheduled notification
- Guard with `Capacitor.isNativePlatform()` so web isn't affected

#### 2. `src/lib/platform.ts` — New utility (tiny)
```typescript
import { Capacitor } from '@capacitor/core'

export const isNative = Capacitor.isNativePlatform()
export const isIOS = Capacitor.getPlatform() === 'ios'
export const isAndroid = Capacitor.getPlatform() === 'android'
export const isWeb = Capacitor.getPlatform() === 'web'
```

#### 3. `src/main.tsx` — Initialize Capacitor plugins on app start
```typescript
import { SplashScreen } from '@capacitor/splash-screen'
import { StatusBar, Style } from '@capacitor/status-bar'
import { isNative } from './lib/platform'

if (isNative) {
  StatusBar.setStyle({ style: Style.Dark })
  SplashScreen.hide()
}
```

#### 4. Safe area padding for iOS notch
Add to `index.css`:
```css
:root {
  --safe-area-top: env(safe-area-inset-top);
  --safe-area-bottom: env(safe-area-inset-bottom);
}
```
Apply to main layout container and bottom nav.

#### 5. `index.html` — Add viewport-fit for iOS
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

#### 6. Wake Lock — Already using Web API, works in Capacitor WebView too. No change needed.

## Risks & Mitigations

**Risk: WebView performance on low-end Android devices.**
Mitigation: BreadBook is a simple UI — text, images, timers. No heavy animations or 3D. Capacitor WebView performance is fine for this. If specific screens feel sluggish, optimize those components.

**Risk: Apple App Store review rejection.**
Mitigation: Common rejection reasons — missing privacy policy, no demo account for reviewers, UI doesn't feel "native enough." Address these proactively: add privacy policy, create a demo account, ensure safe area handling and native-feeling transitions.

**Risk: Maintaining three targets (web, iOS, Android).**
Mitigation: It's one codebase. `npx cap sync` is the only additional step. Platform-specific code is minimal (timer notifications, push registration). The web version is the source of truth.

**Risk: Mac access for iOS builds.**
Mitigation: Andrew has access to a Mac. For CI/CD, GitHub Actions offers macOS runners. Alternatively, services like Codemagic or Appflow can build iOS in the cloud.
